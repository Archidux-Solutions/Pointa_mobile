Page 4 - Prototype & Handoff

Interaction logic, motion intent, and implementation constraints for Flutter.

Prototype flow
Connexion-> Dashboard-> Pointage-> Historique-> Recap-> Sync-> Profil
Flutter handoff tokens
  • Palette: primary/secondary/success/warning/danger + neutral scale
  • Spacing: 8 / 12 / 16 / 24 / 32
  • Radius: 10 / 14 / 20 / 28
  • Typography: Sora (headings) + Plus Jakarta Sans (body)
  • Touch targets: minimum 48dp
UX quality rules
  • Primary CTA always visible on attendance-critical screens
  • Every async error includes clear retry action
  • Offline queue always visible when pending > 0
  • No dead-end states, every screen has next action
  • Adaptive behavior for larger devices/foldables
